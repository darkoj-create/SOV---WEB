SOV v1.0.166 — Hillshade redraw/cache fix

Base used:
- sov-v1.0.166-FULL-wms-cache-thread-tuning.zip

Changed file:
- app/src/main/java/com/darko/speleov1/util/HillshadeTilesOverlay.kt

What changed:
- Hillshade no longer draws z-2, z-1 and current zoom across the full viewport on every frame.
- Lower zoom hillshade tiles are now used only as a per-tile fallback when the current tile is missing.
- Hillshade memory cache is now byte-sized at 32 MB instead of entry-count based.
- Hillshade prefetch was reduced from 2-tile margin / 56 queued tiles to 1-tile margin / 18 queued tiles.
- Hillshade download/cache executor was reduced from 5 threads to 2 threads.
- Map redraw after hillshade tile load is throttled to avoid invalidate storms.

Not changed:
- Hillshade URLs.
- WMS base/overlay URLs.
- Disk cache folder name.
- WMS render logic.
- Existing v1.0.166 WMS cache/thread tuning remains included.

Build note:
- APK inside the original package is not rebuilt in this sandbox because Gradle wrapper cannot download Gradle without internet.
- Open this project in Android Studio and rebuild/sign APK there.
